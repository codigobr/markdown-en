Slack

Slack is one of the most popular messaging applications from internet compatible with some elements of Markdown.

Slack messaging interface

It is fair to say that sending messages on Slack is the tool's most used resource.

Support for Markdown in the Slack messaging interface leaves a little to be desired, however some elements that make up the basic syntax are available to the user.

The application has formatting options in the WYSIWYG style and this is the default form enabled for all users.

The program also has a real-time editor that hides the Markdown formatting of the content while you type the message.

Using the interface formatting controls, in this case, is optional.

To disable the Slack real-time editor, that is, the Parser that hides the Markdown markup, activate the option: Preferences > Advanced > Format messages with markup.

See the image below:

From now on, all Markdown symbols will remain on the screen and the WYSIWYG buttons will also disappear.

Slack allows the user to have several workplaces open at the same time, if this is your case, you must activate Markdown for each one.

You can visit Slack at https://slack.com/.

Support for Markdown in the Slack messaging interface

Table 1.1 shows the Markdown elements that are supported in the Slack message interface, and also provides useful information about some of these elements.

Element Support Information
Titles No
Paragraphs No
Line breaks No Slack does not support the Markdown syntax for breaking a line, but you can still press the key combination Shift + Enter to achieve the same result.
Bold No To format bold text, use an asterisk around the text, example: *Lorem*. Note that this is the standard Markdown syntax for creating content in italics.
Italic Partial Only the use of an underscore is supported, example: _Lorem_.
Blockquote Yes
Ordered lists Yes
Unordered lists Yes
Code Partial Slack's messaging interface does not support code blocks.
Horizontal lines No
Links No
Images No The Slack message editor does not provide support for formatting images in Markdown. You can even drag an image file directly into the interface.
Tables No
Fenced code blocks Yes It is not possible to detach the code within the block.
Footnotes No
Title Id No
Definition lists No
Strikethrough Partial Surround content with just a tilde, example: ~Lorem~.
Task lists No
Emoji (copy and paste) Yes
Emoji (access codes) Yes
automatic URL Link Yes
Disable automatic link from URL Yes
HTML No

It is worth noting that the information above refers to messages used in the Slack user interface, when we use Slack's API for messages we have greater support for elements of the additional Markdown syntax, elements that are not compatible with the user interface .

In the Slack API documentation you can check more information about compatibility.

Slack post interface

The Slack post interface has a Markdown editor in real time, that is, the formatting of the content is shown as you type.

With the real-time editor, the formatting symbols in Markdown are removed and the result is displayed immediately, example: _teste_ --> teste.

To create a new post on Slack, click on the button with a paper clip icon and select the option Post within the menu Create New.
